afterShuffle 时机修复与动画死锁根治 Bugfix Test
6月9日
修复 Start Card shuffle 完成后整个动画系统卡死的严重问题：
  • 死锁根因: shuffle 动画完成后 afterShuffle 立即触发新效果链，但此时旧 chain 尚未关闭，导致 isPlayingEffectAnimations 永远无法重置
  • 修复方案: CombatManager 在 PlayRecorderAnimationsAndWait() 的 finally 块中确保 chain 关闭后再触发 round start / afterShuffle；reveal-zone 移动完成后通过回调精确控制事件时序
  • 测试覆盖: 新增 AfterShuffleTimingTests.cs（190 行），包含 4 个 headless 用例验证 afterShuffle 触发顺序与动画守卫
  • 产出 prd-aftershuffle-reveal-timing-fix-2026-06-08.md 详细记录死锁场景与修复路径

Bury/Stage 反应链动画修复 + 智能 Start Card 排除 Bugfix Feature
6月10日
集中修复 bury/stage 在反应链中的动画丢失与 Start Card 被错误处理的问题：
  • Bury 动画丢失: 当 bury 触发 onMeBuried → Stage 自身时，被 stage 的卡牌因物理索引未刷新而丢失 slot-in 动画；修复方案为在 BuryEffect 中提前 snapshot 并在 RecorderAnimationPlayer 播放前同步物理顺序
  • Self 状态效果动画丢失: 给自身施加状态效果（如 Power）时，若卡牌处于 reveal zone，StatusEffectChange 动画被跳过；通过 StatusEffectGiverEffect.GiveSelfStatusEffect 路径补全捕获
  • 智能 Start Card 排除: BuryEffect 与 StageEffect 在选择目标时自动跳过 isStartCard，避免将 neutral Start Card 埋入牌底或置顶导致流程异常
  • 新增测试卡牌 BUY_ONE_GET_ONE_FREE、CORPSE_CANNON、DOWNED_FIGHTER 验证复杂连锁
