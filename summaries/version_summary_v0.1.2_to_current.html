3. 主要工作方向

3.1 动画系统重构（核心）

  • 从“即时播放”迁移到 EffectRecorder 驱动的逻辑-视觉分离架构：逻辑同步执行，动画在链关闭后批量回放。
  • 新增/完善了 PopUp、SlotIn、PopUpBatch、SlotInBatch、MoveToBottomBatch、MoveToTopPopUpBatch 等成批卡牌位移动画。
  • 新增状态特效抛射物（StatusEffectProjectile），支持按层数飞行、按弹道拆分 delta，并在落地/命中后刷新卡牌文字。
  • Stage、Bury、Exile、AddTempCard、StatusEffectGiver/Consume/Transfer 等效果均接入新动画流程。
  • 修复了 afterShuffle 时序死锁、reactive chain 中 bury→stage 视觉丢失、deck focus 恢复、 reveal-zone race condition 等问题。

3.2 战斗与卡牌机制

  • 新增费用类型：Minion 费用、Bury 费用、Delay 费用、Expose 费用，以及对应的 Cost 实现。
  • “Token”概念统一为“Minion”，支持按类型/拥有者筛选。
  • 新增 takeUpSpace 标记、中立 Start Card 跳过逻辑、敌我卡牌颜色区分、动态伤害描述占位符（<dmg>、<counter>）。
  • 新增状态效果：Counter；完善 Infected VFX、Power Wisp、Status 文本区域。
  • 新增全局战斗动画速度缩放（CombatAnimationSpeed.SpeedScale）和战斗内自动确认（CombatManager.autoReveal）。
  • 新增单局结束检测、整局重置、按 Session 随机敌方卡组池、敌方卡组 HP 加成、商店稀有度权重覆盖。

3.3 代码结构与质量

  • 将大量基于 IntSO 参数的效果重构为 owner / enemy 字段，减少运行时查找。
  • 移除旧的 preEffectEvent、废弃费用系统、Bury/Stage 逻辑阶段同步等旧路径。
  • 输入锁改为引用计数（BlockInput / UnblockInput），并明确职责分离。
  • PhaseManager 在进入商店时重置战斗级 ValueTracker，避免跨局污染。

3.4 视觉与表现

  • 卡牌尺寸、中文字体、9-sliced sprite、圆角 shader、动态阴影、受击反馈、不同所有者颜色。
  • Shuffle 动画节奏优化、卡牌从 deck 到 reveal zone 的 peel/focus 效果、动态弧线中点 z 计算。

3.5 工具、测试与文档

  • 新增多个 Agent Skill：unity-card-design-documenter、unity-card-infinity-check、unity-card-listener-check、unity-card-playmode-test、unity-card-test-planning、unity-onedeck-stats-report、check-default-enemy-deck-pool。
  • 新增大量 headless 测试（bury/stage、afterShuffle、AddTempCard、ConsumeStatus、PowerReact、TransferStatus、敌我视角等）。
  • 新增/更新 Python 工具：prefab 配置检查、卡牌描述与监听器匹配检查、机械分析、测试计划生成、战斗图表等。
  • 补充 CardDesign、TestPlan、RegressionChecklist、VisualBugPrevention 等文档。

4. 关键修复

  • Start Card shuffle 后 afterShuffle 事件与动画死锁。
  • Exile reveal-zone 竞态条件与 takeUpSpace 处理。
  • Bury/Stage 在 recorder 播放期间提前同步 deck 导致的索引错乱。
  • StatusEffect 动画落地前卡牌文字提前提交，造成多层加成显示错误。
  • 卡牌描述中 GameEventListener 响应匹配误报。
  • Cost 失败时的抖动动画未纳入 EffectRecorder 序列。

5. 总结

从 v0.1.2 到当前版本，项目最主要的工作是把战斗动画从即时播放改造为基于 EffectRecorder 的分阶段播放体系，并围绕它补齐了 Bury/Stage/Exile/StatusEffect 等全部效果的视觉表现；同时完成了 Token→Minion、IntSO 参数重构、多类新费用/新事件/新状态效果，以及商店/单局循环的元游戏支持。此外，测试、文档和 Agent Skill 也得到了系统性补充。
