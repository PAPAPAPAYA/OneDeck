Pop Up + Slot In 动画设计

让卡组中的每一张关键卡片都能被玩家清晰看见

🎯设计目标

当前战斗中，卡片层层堆叠，当卡片被添加、放逐、获得或失去状态效果时，玩家往往无法一眼看出"到底是哪张卡"受到了影响。 本方案引入 Pop Up（弹起） + Slot In（插回） 动画组合， 让关键卡片在受影响的瞬间主动"跳"到视觉 foreground，展示完毕后再优雅地回到卡组原位。

🎬四个使用场景

场景 A

卡片被添加到卡组

当效果向卡组中添加临时卡片时，新卡先 pop up 展示自己，让玩家知道"来了什么卡"，然后再 slot in 进入堆叠。

Pop Up → 展示 0.2s → Slot In
场景 B

卡片被放逐

放逐发生时，目标卡先 pop up 让玩家看清"什么卡没了"，然后再执行退场销毁动画。

Pop Up → 展示 0.1s → Destroy 退场
场景 C

状态效果投射物

当诅咒/力量等投射物飞向目标卡时，目标卡 pop up，在高位接收投射物，投射物结束后 slot in 回到卡组。

Pop Up → Projectile 飞行 → Tint 变色 → Slot In
场景 D

状态效果被消耗

当卡片上的状态效果（如 Mana、Power）被消耗时，该卡 pop up 让玩家看清"哪张卡的什么效果被消耗了"，然后 slot in。

Pop Up → Tint 消退 → Slot In

⚙️动画参数配置

以下参数可在 Unity Inspector 中实时调整，位于 CombatUXManager 组件。

参数名 类型 默认值 说明
popUpYOffset float 1.5 垂直弹起高度（世界单位）
popUpZBoost float -1.0 向镜头方向偏移（负值 = 更靠前/更可见）
popUpScaleMultiplier float 1.15 弹起时的缩放倍率（1.15 = 放大 15%）
popUpDuration float 0.25s 弹起动画时长
popUpEase Ease OutQuad 弹起缓动曲线（快起慢停）
slotInDuration float 0.35s 插回动画时长
slotInEase Ease InOutQuad 插回缓动曲线（平滑起降）

📋关键设计决策

问题 决策
所有加卡都用 PopUp+SlotIn 吗？ 只有战斗中临时添加（AddTempCard）使用。战斗初始化（InstantiateAllPhysicalCards）保持 instant，避免开局过慢。
Reveal Zone 的卡片需要 PopUp 吗？ 不需要。揭示区卡片本身已在前景，无需额外弹起。
PopUp 期间卡组 Peel/Focus 怎么办？ RecorderAnimationPlayer 在播放 PopUp/SlotIn 前，若检测到 Deck Focus 状态，先执行 RestoreDeckFocusCoroutine。
Headless 测试环境怎么办？ NullCombatVisuals 中 PopUp/SlotIn 为空实现（no-op），逻辑层不受影响。
需要加音效吗？ 本期 PRD 不包含音效。如需可在 CombatUXManager.PopUpCard 中后续扩展。
